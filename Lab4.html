Lab 4: Lập trình NFT và Dapps (NT547)

Xây dựng ứng dụng phi tập trung End-to-End

Mục tiêu và Tổng quan

Bài lab tổng hợp này được thiết kế để cung cấp cho sinh viên một cái nhìn toàn diện về quy trình phát triển một sản phẩm blockchain hoàn chỉnh, từ "backend" (on-chain) đến "frontend" (off-chain).

  • Mục tiêu (Backend): Hiểu và áp dụng tiêu chuẩn ERC-721 để lập trình một hợp đồng thông minh (Smart Contract) cho bộ sưu tập NFT. Sinh viên sẽ học cách xử lý metadata, lưu trữ trên IPFS và sử dụng thư viện OpenZeppelin.
  • Mục tiêu (Frontend): Tích hợp Smart Contract đã tạo với một giao diện người dùng (React.js) để xây dựng một ứng dụng phi tập trung (DApp) hoàn chỉnh. Sinh viên sẽ học cách dùng Ethers.js để kết nối frontend với blockchain qua ví MetaMask.
  • Tổng quan: Đây là bài lab then chốt, kết nối hai thế giới: logic on-chain (Solidity) và giao diện off-chain (JavaScript/React). Việc nắm vững kỹ năng này là điều kiện tiên quyết để tạo ra các sản phẩm blockchain mà người dùng cuối có thể tương tác.

Phần 1: Xây dựng Smart Contract (Backend On-chain)

Trong phần này, chúng ta sẽ tập trung vào việc tạo ra một Smart Contract tuân thủ tiêu chuẩn ERC-721.

1.1. Kiến thức Nền tảng (On-chain)

ERC-721 (Non-Fungible Token)

Đây là tiêu chuẩn cho các token không thể thay thế cho nhau. Mỗi token là duy nhất, có một định danh tokenId riêng và có thể có các thuộc tính khác nhau. Đây là tiêu chuẩn cho vật phẩm sưu tầm, nghệ thuật số, v.v.

Metadata và IPFS

Mỗi NFT gắn liền với dữ liệu mô tả (tên, hình ảnh, thuộc tính) gọi là metadata. Lưu trữ hình ảnh trực tiếp lên blockchain là cực kỳ tốn kém. Do đó, chúng ta sẽ:

  1. Lưu trữ hình ảnh và file metadata (JSON) trên IPFS (InterPlanetary File System) - một hệ thống lưu trữ phi tập trung.
  2. Lưu một liên kết (URI) trỏ đến file metadata đó trên blockchain.
  3. Chúng ta sẽ sử dụng dịch vụ "pinning" như Pinata.cloud để đảm bảo dữ liệu luôn có sẵn.

OpenZeppelin

Một thư viện mã nguồn mở cung cấp các Smart Contract đã được kiểm toán bảo mật. Thay vì tự viết lại toàn bộ logic ERC-721, chúng ta sẽ kế thừa từ các hợp đồng của OpenZeppelin. Đây là một "best practice" trong ngành.

1.2. Hướng dẫn Thực hành (Phần 1)

Bước 1.1: Chuẩn bị và Tải Metadata lên IPFS

  1. Chuẩn bị 1 file hình ảnh bất kỳ (.png, .jpg) cho NFT của SV.
  2. Đăng ký tài khoản miễn phí trên Pinata.cloud.
  3. Tải file hình ảnh của SV lên Pinata. Sau khi thành công, sao chép mã CID của hình ảnh (ví dụ: Qm...).
  4. Tạo một file văn bản mới tên là mynft.json với nội dung sau, thay thế YOUR_IMAGE_CID bằng mã CID SV vừa sao chép: 
    {
    "name": "My First DApp NFT",
    "description": "A unique collectible from the combined lab.",
    "image": "ipfs://YOUR_IMAGE_CID"
}
  5. Tải file mynft.json này lên Pinata.
  6. Sau khi tải lên, sao chép CID của file JSON. URL đầy đủ trỏ đến metadata của SV sẽ là:
    https://gateway.pinata.cloud/ipfs/YOUR_METADATA_CID

Quan trọng: Hãy lưu lại URL metadata này, chúng ta sẽ cần nó ở cả Phần 1 và Phần 2.

Bước 1.2: Viết Hợp đồng NFT (MyNFT.sol)

  1. Mở Remix IDE, tạo file mới MyNFT.sol trong thư mục contracts.
  2. Dán đoạn mã sau vào file:
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

// Import các hợp đồng cần thiết từ thư viện OpenZeppelin
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721URIStorage.sol";
import "@openzeppelin/contracts/access/Ownable.sol";
import "@openzeppelin/contracts/utils/Counters.sol";

// Hợp đồng của chúng ta kế thừa từ các hợp đồng của OpenZeppelin
contract MyNFT is ERC721URIStorage, Ownable {

    // Sử dụng thư viện Counters để tạo tokenId một cách an toàn
    using Counters for Counters.Counter;
    Counters.Counter private _tokenIds;

    // Constructor: được gọi khi hợp đồng được triển khai
    // Nó khởi tạo bộ sưu tập NFT với tên và ký hiệu
    constructor() ERC721("MyNFTCollection", "MNC") Ownable(msg.sender) {}

    // Hàm để mint một NFT mới
    // Chỉ "owner" của hợp đồng mới có thể gọi hàm này
    function safeMint(address to, string memory uri) public onlyOwner {
        // Lấy tokenId hiện tại và tăng nó lên cho lần mint tiếp theo
        uint256 tokenId = _tokenIds.current();
        _tokenIds.increment();

        // Mint token mới và gán cho địa chỉ "to"
        _safeMint(to, tokenId);

        // Đặt URI metadata cho token mới được mint
        _setTokenURI(tokenId, uri);
    }
}

Bước 1.3: Biên dịch và Triển khai

  1. Chuyển đến tab Solidity Compiler, chọn phiên bản compiler 0.8.20 (hoặc mới hơn) và nhấn Compile MyNFT.sol.
  2. Chuyển đến tab Deploy & Run Transactions.
  3. Chọn Environment là Injected Provider - MetaMask. Đảm bảo MetaMask của SV đang kết nối với mạng Sepolia Testnet.
  4. Nhấn Deploy và xác nhận giao dịch trên MetaMask.

Quan trọng: Sau khi triển khai thành công, hãy sao chép Địa chỉ Hợp đồng (Contract Address) của SV và lưu lại. SV sẽ cần nó cho Phần 2.

Bước 1.4: Kiểm tra Hợp đồng (Mint trên Remix)

Trước khi xây dựng DApp, chúng ta cần đảm bảo hợp đồng hoạt động chính xác.

  1. Trong Remix, bên dưới mục "Deployed Contracts", tìm hợp đồng MyNFT của SV.
  2. Mở rộng nó, SV sẽ thấy hàm safeMint.
  3. Trong trường to (address): dán địa chỉ ví MetaMask của SV.
  4. Trong trường uri (string): dán URL metadata từ Pinata (Bước 1.1).
  5. Nhấn transact và xác nhận giao dịch trên MetaMask.

Bước 1.5: Xác thực trên OpenSea

  1. Truy cập https://testnets.opensea.io/.
  2. Kết nối ví MetaMask của SV.
  3. Đi đến trang cá nhân (Profile). SV sẽ thấy NFT mình vừa mint xuất hiện. (Có thể mất vài phút để OpenSea cập nhật).

Nếu SV thấy NFT của mình, xin chúc mừng! "Backend" on-chain đã hoạt động hoàn hảo. Giờ hãy xây dựng giao diện cho nó.

Phần 2: Xây dựng Giao diện DApp (Frontend Off-chain)

Ở phần này, chúng ta sẽ tạo một trang web React đơn giản để người dùng có thể kết nối ví và mint NFT từ chính hợp đồng SV vừa triển khai.

2.1. Kiến thức Nền tảng (Off-chain)

Kiến trúc DApp

Kiến trúc DApp khác với web truyền thống: Frontend (React) -> Wallet (MetaMask) -> Blockchain (Ethereum). Ví MetaMask đóng vai trò trung gian xác thực và ký giao dịch thay cho máy chủ backend.

Node.js và React

  • Node.js: Môi trường thực thi JavaScript, cần thiết để chạy các công cụ xây dựng dự án React.
  • React: Thư viện JavaScript phổ biến để xây dựng giao diện người dùng. Chúng ta sẽ dùng create-react-app để tạo dự án.

Ethers.js

Thư viện JavaScript mạnh mẽ để tương tác với blockchain Ethereum. Ba khái niệm cốt lõi:

  • Provider: Đại diện cho kết nối đến mạng Ethereum (ví dụ: qua MetaMask) để đọc dữ liệu.
  • Signer: Đại diện cho một tài khoản người dùng (lấy từ Provider) có thể ký giao dịch và thay đổi trạng thái (ghi dữ liệu).
  • Contract: Một đối tượng JavaScript đại diện cho Smart Contract của SV trên blockchain. SV khởi tạo nó với (1) Địa chỉ, (2) ABI, và (3) Signer.

2.2. Hướng dẫn Thực hành (Phần 2)

Bước 2.1: Lấy ABI và Địa chỉ Hợp đồng

  1. Địa chỉ Hợp đồng: Lấy địa chỉ SV đã lưu ở Bước 1.3.
  2. Lấy ABI:
    • Quay lại Remix IDE, tab Solidity Compiler.
    • Sau khi biên dịch thành công, nhấn nút "ABI" để sao chép toàn bộ ABI (dưới dạng JSON).

Bước 2.2: Thiết lập Dự án React

  1. Mở terminal (dòng lệnh) trên máy tính của SV.
  2. Chạy các lệnh sau để tạo dự án React mới và cài đặt Ethers.js: 
    npx create-react-app my-nft-dapp
cd my-nft-dapp
npm install ethers@5.7
  3. Mở thư mục my-nft-dapp trong trình soạn thảo code (ví dụ: VS Code).
  4. Tạo một thư mục mới src/utils. Bên trong, tạo file MyNFT.json và dán toàn bộ ABI SV đã sao chép ở trên vào file này.

Bước 2.3: Xây dựng Giao diện và Kết nối MetaMask

Mở file src/App.js, xóa toàn bộ nội dung mặc định và thay thế bằng mã sau. Đừng quên cập nhật 2 giá trị quan trọng!

// src/App.js
import { useState } from 'react';
import { ethers } from 'ethers';
import MyNFT_ABI from './utils/MyNFT.json';
import './App.css';

// ==================================================================
// !!! CẬP NHẬT 2 GIÁ TRỊ NÀY !!!
// Dán địa chỉ hợp đồng của SV từ Bước 1.3
const contractAddress = "YOUR_NFT_CONTRACT_ADDRESS"; 
// Dán URL metadata của SV từ Bước 1.1
const metadataURI = "https://gateway.pinata.cloud/ipfs/YOUR_METADATA_CID";
// ==================================================================

function App() {
  const [walletAddress, setWalletAddress] = useState(null);
  const [mintingStatus, setMintingStatus] = useState(null);
  const [txHash, setTxHash] = useState(null);

  async function connectWallet() {
    if (typeof window.ethereum !== 'undefined') {
      console.log('MetaMask is installed!');
      try {
        const accounts = await window.ethereum.request({ method: 'eth_requestAccounts' });
        console.log('Connected account:', accounts[0]);
        setWalletAddress(accounts[0]);
      } catch (error) {
        console.error('User denied account access', error);
      }
    } else {
      console.log('Please install MetaMask!');
      alert('Please install MetaMask!');
    }
  }

  async function handleMint() {
    if (!walletAddress) {
      alert("Please connect your wallet first.");
      return;
    }
    
    if (typeof window.ethereum !== 'undefined') {
      const provider = new ethers.providers.Web3Provider(window.ethereum);
      const signer = provider.getSigner();
      // Khởi tạo đối tượng Contract
      const nftContract = new ethers.Contract(contractAddress, MyNFT_ABI, signer);

      try {
        setMintingStatus('Minting... Vui lòng xác nhận giao dịch trên MetaMask.');
        setTxHash(null);

        // Gọi hàm safeMint từ Smart Contract
        const tx = await nftContract.safeMint(walletAddress, metadataURI);
        console.log('Transaction sent:', tx.hash);
        
        setMintingStatus('Đang chờ giao dịch được xác nhận...');

        // Chờ giao dịch được xác nhận
        await tx.wait();
        
        console.log('Transaction confirmed!');
        setMintingStatus('Mint thành công!');
        setTxHash(tx.hash);

      } catch (error) {
        console.error('Minting failed:', error);
        setMintingStatus(`Mint thất bại: ${error.message}`);
      }
    }
  }

  // Giao diện cơ bản (SV có thể tự làm đẹp bằng CSS/Tailwind)
  return (
    <div className="App">
      <header className="App-header">
        <h1>My NFT DApp</h1>
        
        <button onClick={connectWallet} className="connect-button">
          {walletAddress 
            ? `Connected: ${walletAddress.substring(0, 6)}...${walletAddress.substring(38)}` 
            : 'Connect Wallet'}
        </button>

        {walletAddress && (
          <div className="mint-container">
            <button onClick={handleMint} className="mint-button">
              Mint My NFT
            </button>
            
            {mintingStatus && <p className="status-text">{mintingStatus}</p>}
            
            {txHash && (
              <p className="tx-link"&Sgt;
                Xem giao dịch: 
                <a href={`https://sepolia.etherscan.io/tx/${txHash}`} target="_blank" rel="noopener noreferrer">
                  Etherscan
                </a>
              </p>
            )}
          </div>
        )}
      </header>
    </div>
  );
}

export default App;

Phần 3: Yêu cầu và Bài tập Nâng cao

Bây giờ, hãy kết hợp và nâng cấp cả backend lẫn frontend với các tính năng mới.

  1. Nâng cấp Smart Contract (Backend):

    Sửa đổi hợp đồng MyNFT.sol để thêm 2 tính năng:

    • Thêm Phí Mint: Yêu cầu người gọi phải trả một khoản phí (ví dụ: 0.01 ETH) cho mỗi lần mint.
      Gợi ý: Thêm payable vào hàm safeMint, khai báo biến mintFee và dùng require(msg.value >= mintFee, ...).
    • Giới hạn Tổng cung (Max Supply): Chỉ cho phép mint một số lượng NFT tối đa (ví dụ: 100).
      Gợi ý: Khai báo biến MAX_SUPPLY và kiểm tra require(_tokenIds.current() < MAX_SUPPLY, ...).

    Sau khi sửa, SV phải triển khai lại hợp đồng và cập nhật contractAddress mới trong DApp.

  2. Cập nhật DApp (Frontend) để hỗ trợ Phí Mint:

    Sửa file App.js để DApp có thể hoạt động với hợp đồng mới. Khi gọi hàm safeMint, SV cần gửi kèm giá trị ETH (phí mint).

    Gợi ý: Với Ethers.js, SV truyền giá trị ETH bằng cách thêm một đối tượng override vào cuối lệnh gọi hàm: 

    const mintFee = ethers.utils.parseEther("0.01");
const tx = await nftContract.safeMint(walletAddress, metadataURI, { 
    value: mintFee 
});

  3. Hiển thị Thông tin NFT (Frontend):

    Xây dựng một giao diện (ô input, nút) cho phép người dùng nhập vào một tokenId. Khi nhấn nút, DApp sẽ:

    1. Gọi hàm tokenURI(tokenId) trên hợp đồng (đây là hàm view, không tốn gas).
    2. Sau khi nhận được URI, dùng fetch API của JavaScript để lấy nội dung file JSON metadata từ IPFS.
    3. Hiển thị tên, mô tả, và hình ảnh của NFT đó lên giao diện.
  4. Hiển thị Bộ sưu tập của Người dùng (Frontend - Nâng cao):

    Thêm chức năng hiển thị tất cả các NFT mà địa chỉ ví đang kết nối sở hữu.

    Gợi ý: Đây là một yêu cầu phức tạp. Cách làm chuẩn là SV phải nâng cấp hợp đồng (Phần 1) để kế thừa thêm ERC721Enumerable.sol của OpenZeppelin. Tiêu chuẩn mở rộng này sẽ cung cấp các hàm balanceOf(owner) để lấy số lượng NFT và tokenOfOwnerByIndex(owner, index) để lấy từng tokenId. Sau đó, SV lặp qua các tokenId để lấy tokenURI và hiển thị chúng.

  5. Viết Báo cáo Tổng kết (Đặt tên MSSV.zip, nộp tại trang web môn học):

    Nộp báo cáo bao gồm:

    • Toàn bộ mã nguồn MyNFT.sol (đã nâng cấp) và dự án React (dưới dạng file zip hoặc link Github).
    • Địa chỉ hợp đồng đã triển khai trên Sepolia.
    • Link đến bộ sưu tập của SV trên Testnets OpenSea.
    • Mô tả chi tiết logic và các đoạn code SV đã thêm/sửa để hoàn thành các bài tập nâng cao.
    • Chụp ảnh màn hình DApp của SV đang hoạt động với các tính năng mới.
